# Web 实时配置 (Web Configuration)

MCP-ANY-REST 提供了一个基于 Web 的实时配置界面，允许你在运行时动态修改服务器配置、管理模块和应用预设。

## 访问方式

Web 配置界面默认集成在服务器中，可以通过浏览器访问服务器的根路径：

```
http://localhost:<port>/
```

其中 `<port>` 是你在配置文件中设置的服务器端口（默认为 3000）。

## 主要功能

### 1. 概览与状态
- 查看当前服务器的运行状态。
- 查看基础配置信息（Base URL, Log Level 等）。

### 2. 模块管理 (Modules)
- **启用/禁用模块**：实时开启或关闭特定功能模块。
- **访问控制**：设置模块的访问级别（Public, Protected, Private）。
- **速率限制**：配置每分钟/每小时的请求限制及突发容量。
- **方法级控制**：深入到具体方法（Method）级别的精细化配置。

### 3. 预设管理 (Presets)
- **应用预设**：快速切换不同的配置预设（如 "Full", "Minimal"）。
- **保存预设**：将当前配置保存为新的预设。
- **加载/删除**：管理已保存的配置快照。

### 4. 服务器配置
- 修改服务器端口。
- 调整日志级别。
- 更新 API Base URL。

### 5. 认证配置 (Authentication)
- 切换认证模式（Direct / Login）。
- 配置认证策略（Bearer Token, API Key, OAuth2 等）。
- 设置 Token 过期时间和刷新策略。

## 使用说明

所有的配置更改都是即时生效的（Dynamic Configuration），通常无需重启服务器。配置更改会自动持久化到配置文件中，确保下次启动时保持一致。
